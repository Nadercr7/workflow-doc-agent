Architecture

The agent loop

Three stages, each one validated by a Pydantic model. Every arrow is typed. Every LLM call returns a validated instance or raises. Nothing parses raw strings outside the providers themselves.

In

Folder of work

One .py file plus one .xlsx file. Discovered with discover_workflow_files().

↓
1

readers.py · pre-digest

AST parse for the Python file (imports, functions, classes, top-level calls). openpyxl read-only for the workbook (sheets, headers, sample rows). Frozen dataclasses; no model sees the raw file.

↓
2

providers.parse(output_format=WorkflowSummary)

Cheap stage. Returns a validated WorkflowSummary with three to five clarifying_questions. Models: gemini-2.5-flash or claude-haiku-4-5.

↓
3

collect_answers

Human in the loop with Rich prompts, or --non-interactive for unattended runs. Result is a WorkflowAnswers object, one answer per question id.

↓
4

providers.parse(output_format=FinalDoc)

Step-up stage. Takes summary plus answers, returns a FinalDoc. Models: gemini-2.5-pro or claude-sonnet-4-5.

↓
Out

FinalDoc.to_markdown() · written to disk

Lands at outputs/<workflow>_<provider>.md. Sections: Overview, Schedule, Inputs, Outputs, Runbook, Open Questions. Ready to publish.

Schemas

Every output is a Pydantic model

Model Stage Notes
PythonFileSummary internal One-line purpose, inputs, outputs, deps
ExcelFileSummary internal Sheets, columns, sample-row guesses
WorkflowSummary stage 1 output Includes 3 to 5 clarifying_questions
WorkflowAnswers human input One Answer per Question.id
FinalDoc stage 2 output Has to_markdown() for the published file
Live demo path

Gemini provider

Schema is rendered into the system prompt with response_mime_type="application/json". The text response is parsed with model_validate_json(). Pydantic emits keys Gemini's validator rejects, so we strip those for the prompt copy and validate client-side. Same guarantee, fewer moving parts. 

resp = client.models.generate_content(
    model=model,
    contents=user,
    config=GenerateContentConfig(
        system_instruction=full_system,
        response_mime_type="application/json",
        max_output_tokens=max_tokens,
    ),
)
parsed = output_format.model_validate_json(resp.text)
Drop-in swap

Claude provider

Each Pydantic schema becomes the input_schema of a single forced tool call. Claude is required to call that tool, so the response is structured by construction. Works across all current Claude models. 

resp = client.messages.create(
    model=model,
    tools=[{
        "name": tool_name,
        "input_schema": output_format.model_json_schema(),
    }],
    tool_choice={"type": "tool", "name": tool_name},
    messages=[{"role": "user", "content": user}],
)
parsed = output_format.model_validate(resp.content[0].input)
Failure handling

What happens when things go wrong

API transient errors

Timeouts, 5xx, rate-limits. Retried with exponential backoff in retry.py.

Budget exceeded

The next call after the running total clears the budget raises BudgetExceeded. Run halts.

Schema mismatch

model_validate_json raises with the full validation error, pointing at the exact field.

Code-fenced JSON from Gemini

Stripped automatically by _strip_code_fences before retry.
