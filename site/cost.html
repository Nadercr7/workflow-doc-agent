Cost & evals

Designed to scale to many, many workflows

A single run is cheap by accident. A thousand runs a month are cheap by design. The cost tracker, the model routing, and the eval harness exist so the second number stays a true claim instead of a hopeful one.

Routing

Cheap stages run on the small model. Final-doc generation steps up to the larger model. Most workflows only need the cheap path; the upgrade is one-line revertible. 

_ROUTING = {
  ("gemini", "summary"):    "gemini-2.5-flash",
  ("gemini", "questions"):  "gemini-2.5-flash",
  ("gemini", "final_doc"):  "gemini-2.5-pro",
  ("claude", "summary"):    "claude-haiku-4-5",
  ("claude", "questions"):  "claude-haiku-4-5",
  ("claude", "final_doc"):  "claude-sonnet-4-5",
}

Hard ceiling

Budget enforcement that actually fires

A run that would have cost $1.20 stops at the first call past the budget, not after burning all $1.20. The tracker is a frozen dataclass, so every call is in an immutable audit trail by default. 

@dataclass(frozen=True)
class CostTracker:
    budget_limit_usd: float
    records: tuple[CostRecord, ...] = ()

    def add(self, record: CostRecord) -> "CostTracker":
        return CostTracker(
            self.budget_limit_usd,
            self.records + (record,),
        )

# in the agent loop
if cost.over_budget:

Eval-driven development

Two evals, both runnable in seconds

capability_eval.py

Pass/fail checks on the generated Markdown. Currently 9 of 9 on the finance sample.

  • File exists
  • Has H1 title
  • Has Overview, Schedule, Inputs, Outputs, Runbook, Open Questions
  • Length is non-trivial (≥ 600 chars)

regression_eval.py

Diffs the latest output against a checked-in baseline. Today it's a placeholder that compares baseline against itself.

In CI, it gates merges: a prompt edit that drops a Pydantic field or rewrites a section into something shorter and worse fails the diff.
