An agent that documents production workflows, before anyone has to read the code.

Point it at a folder with a Python file and an Excel report. It pre-digests both, asks three to five clarifying questions about schedule and ownership, and writes a publishable Markdown runbook. The same loop runs on Gemini or Claude with one env var. Half a cent per workflow.

From clone to first generated doc
01 / read

Read the folder safely

The Python file is parsed with ast, the Excel file is opened with openpyxl in read-only mode. No code is executed, no full file is dumped to the model. Only structured digests cross the wire.

02 / summarize and ask

Cheap model does the extraction

A small model (Flash or Haiku) returns a Pydantic-validated summary plus three to five clarifying questions about schedule, owner, and downstream consumers. Wrong shape fails fast, before any expensive call is made.

03 / document

Step-up model writes the runbook

A larger model (Pro or Sonnet) takes the summary plus your answers and emits a FinalDoc with Overview, Schedule, Inputs, Outputs, Runbook, and Open Questions. Rendered to Markdown, ready to publish.

Quickstart

From zero to a generated doc in five minutes

git clone https://github.com/Nadercr7/workflow-doc-agent
cd workflow-doc-agent
py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
copy .env.example .env
# put GEMINI_API_KEY=... or ANTHROPIC_API_KEY=... in .env

python samples\revenue_forecast\forecast_pipeline.py
workflow-doc run samples\revenue_forecast --provider gemini --budget 0.50 --non-interactive
# -> outputs/revenue_forecast_gemini.md
Provider abstraction

One interface, two implementations

The agent never imports anthropic or google.genai directly. Both providers satisfy a small LLMProvider Protocol. Switching is one env var.

Vendor lock-in is a real cost. The abstraction is cheap, and it means the day Anthropic raises prices or Google ships a faster Flash, the same agent code keeps working.

    name: str

Production fit

Designed to slot in, not to be the centerpiece

Dagster trigger

Run run_agent per workflow folder on a cron, or per file change.

Box / Drive watcher

A new .py or .xlsx kicks off a fresh run, and the cost ceiling keeps the bill bounded.

Notion / GitHub publish

The agent emits Markdown. Publishing is a separate, swappable adapter. Notion, GitHub, Confluence, all the same to the agent.
